MY FIRST TASK

Juniper Mist VC, Feb 14-15
Assignment

Replace an old Avaya and 3Com switch with three Juniper switches that are joined in a virtual chassis (VC). The configuration has to be managed by Mist.

Work Process

I created a clear guideline, in the form of a spreadsheet, for the configuration of the VC ports. I started by mapping the old ports to the ones of the VC. After that I made an overview of the VLANs and the port and IP configurations on the old switches.

Although I had never used Mist AI before, the way it worked was not a mystery to me because of the bootcamp I had followed the week before.
Soulaiman had already claimed the switches and added them to a VC so I could immediately start with the configuration.
I configured the IP settings and created the VLANs. After linking the VLANs to port profiles, I could finally use these to create port configurations.

This task was assigned to me by Yves, on wednesday the 23th of February he gave me feedback on the configuration.

PALO ALTO FIREWALL

pa-460 firewall deploy, Mar 8
Assignment

Replace with two pa-320 firewalls with the newer pa-460 version and transfer the config.

Access the web interface

By default the only way to configure the firewall is by connecting to the console. To access the web interface, ZTP needs to be disabled and an IP needs to be assigned to the management port.

Configuration

The configuration from the old firewalls can be exported as an XML file. This file can be uploaded, staged and committed on one of the new firewalls. As the two switches were linked, they were automatically added to a cluster by the old configuration.

Installation

The firewalls will be mounted in their racks on the 24th of March.


RAID FALURE RECOVERY

recover development environment, Mar 10,14,15
Problem

On the 9th of March, two of the four disks in the RAID10 array failed at the same time, presumeably due to a power surge. In this usecase, RAID10 allows for failure of two drives. However the failing drives were in the same RAID1 pool, preventing the array from being rebuilt and making all data unrecoverable.

Repairing Hardware

The first step in rebuilding the development environment was replacing the broken drives with identical new ones. The raid controller was later used to create a new RAID array.

Recovering Data

We mounted the new array in ESX and recreated the XSOAR, SilverFort and Palo Alto firewall VMs. I reinstalled and initiated some integrations from the marketplace on the XSOAR server and uploaded playbooks from backups.
