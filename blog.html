MY FIRST TASK

Juniper Mist VC, Feb 14-15
Assignment

Replace an old Avaya and 3Com switch with three Juniper switches that are joined in a virtual chassis (VC). The configuration has to be managed by Mist.

Work Process

From the configuration files of the old switches I made an spreadsheet of all the VLANs, port configurations and the IP configuration of the switch.
Next, I mapped the old ports to ports on the VC. Now I had created a clear guidline for the configuration of the VC ports.

Although I had never used Mist AI before, the way it worked was not a mystery to me because of the bootcamp I had followed the week before.
Soulaiman had already claimed the switches and added them to a VC so I could immediately start with the configuration.
I configured the IP settings and created the VLANs. After linking the VLANs to port profiles, I could finally use these to create port configurations.

Evaluation

This task was assigned to me by Yves, on wednesday the 23th of February he gave me feedback on the configuration.

AUTOMATED EMAILS WITH CORTEX XSOAR

XSOAR data collection email and HTML messaging template, Feb 22
Assignment

This task has two parts, the first is to create a CSS template in XSOAR that is used by the automatic Emails of playbooks.
Secondly I had to edit several playbooks to use Keys, variables within XSOAR, that can be reused by several mailing tasks within the playbook. Before these tasks used similar blocks of HTML that had to be written individually for every mailing task.

Content of the mail

By adding "set" tasks to the playbook I created the necessary keys and as the playbook progressed this kind of task was also used to concatinate more HTML to the key.
Finally the mailing task could use these keys as content of the Email with the syntax ${KeyName}.

Layout of the mail

In Cortex XSOAR you can add a server configuration that contains a style template for all mails this server will send.
You can apply it by adding a server configuration named "messages.html.formats.externalformsubmit" under Settings > About > Troubleshooting.
